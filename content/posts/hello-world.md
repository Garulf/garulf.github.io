Hi,

I created this site to represent the passion I have for my projects. I love designing code and felt I needed a place to show off my work and a space that allows me to easily create more.

I felt that it was only right for my site to reflect where I spend a lot of my time with my code—the terminal. I am still working on the UX and rounding out any rough edges. The goal is to make the site usable for terminal pros and those who have never even seen a command line. Navigating by directory seems to work okay, but allowing the user to navigate after reading a section can be tricky. To solve this, I added a command suggestion feature that lets users bring up navigation on demand. Previous iterations had navigation menus pop up automatically after every section, but it stole too much control away from the user and became a distraction.

Currently, only the last 4 blog posts are shown on the main view. I am testing different avenues of exposing older posts while staying true to the terminal aesthetic—whether that's implementing a classic less or more pagination style, or a dedicated history command.

This site is very much a living project, and I’ll be tweaking the interface as I find better ways to balance CLI authenticity with a smooth web experience. If you happen to break something or find a clunky interaction while exploring, feel free to drop me some feedback.

Thanks

Garulf
